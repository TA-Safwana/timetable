## Architecture

**Stack:** Vite + React 19, Firebase (Auth + Firestore), react-router-dom v7, vite-plugin-pwa (Workbox).

**Auth flow:** `useAuth` wraps `onAuthStateChanged` and returns `undefined` (loading) / `null` (signed out) / user object. `RequireAuth` in `App.jsx` gates all routes except `/login` and `/share/:shareId`. The public share route receives `user` as a prop (may be null) so it can conditionally show the import CTA.

**Firestore data model:**
```
users/{uid}/children/{childId}
  name, emoji, className, order, hasSaturday, shareId

users/{uid}/children/{childId}/timetable/{dayKey}
  subjects: string[]          // dayKey = mon|tue|wed|thu|fri|sat

sharedTimetables/{shareId}    // public snapshot, no auth required
  ownerUid, childName, childEmoji, childClass, hasSaturday,
  timetable: { mon: [], tue: [], ... }, createdAt
```

**Real-time hooks:** `useChildren` and `useTimetable` use `onSnapshot` for live updates. `useSharedTimetable` uses a one-time `getDoc`. All hooks return sensible defaults (`[]` / `{}`) while loading — screens never need to handle undefined data.

**Routing:** All routes except `/share/:shareId` require auth. Route params (`childId`, `shareId`) are extracted in wrapper components in `App.jsx` via `useParams()` and passed as props to screens — screens themselves don't call `useParams`.

**Packing state:** The "Pack" button state on the home screen is stored in `sessionStorage` keyed by `packed:{childId}:{day}:{YYYY-M-D}`. It auto-clears the next day and when the tab closes. No Firestore writes.

**Share flow:** `ShareScreen` creates a snapshot in `sharedTimetables/{nanoid(10)}` and writes the `shareId` back to the child doc. `SharedTimetableScreen` reads it publicly. Post-sign-in redirect uses `sessionStorage('postShareImport')`.

## Design system

CSS custom properties in `src/styles/global.css`. Key values:
- `--radius: 10px`, `--radius-pill: 999px`
- `--primary: #EF2D56`, `--secondary: #2FBF71`, `--accent: #ED7D3A`
- Palette also includes `#8CD867` (green-light) and `#363537` (dark text base)
- `--text: #1a1a1a`, `--text-muted: #666`

**Color palette** (from brand): `#363537`, `#EF2D56`, `#ED7D3A`, `#8CD867`, `#2FBF71`. No yellow — Sunday/Saturday rest cards use the green family.

**Cards:** White background, `box-shadow: 0 2px 8px rgba(0,0,0,0.06)`, colored left border via `borderLeft: '5px solid {accent}'`. Subject/tip cards follow this pattern consistently.

**Day colors:** Defined in `DAY_COLORS` in `src/components/DayNav.jsx` — each day has `{ bg, text, border }`. Text colors are dark variants (e.g. `#b01c39` not `#EF2D56`) to meet WCAG AA contrast on their respective light backgrounds.

**Subject palette:** In `BookList.jsx` and `SubjectTag.jsx`, colors are assigned by **index position** (not name hash) so adjacent subjects always get visually distinct colors.

## PWA

Icons must exist at `public/icons/icon-192.png` and `public/icons/icon-512.png`. The service worker is auto-generated by Workbox on `npm run build`. For Netlify deployment, add a `netlify.toml` with SPA rewrite (`/* → /index.html`) and set all `VITE_*` env vars in the Netlify dashboard. Add the Netlify URL to Firebase → Authentication → Authorized Domains.
